====================================================
MANDATORY DRILL-DOWN UI PHILOSOPHY
====================================================

The entire application MUST follow a deep drill-down operational monitoring philosophy.

This is a mandatory core UI/UX requirement.

At any and every meaningful dashboard card, KPI, table row, metric, alert, report item, graph, counter, summary, or entity:
the user must be able to click and drill down deeper into the underlying data.

The platform should NEVER show dead/static summary cards wherever drill-down is possible.

Every click should progressively move from:
summary
→ category
→ subcategory
→ entity
→ detailed record
→ exact data entry/log/event/task.

Examples:

1. Academic KPI Flow:
Classes Behind Syllabus
→ Class
→ Section
→ Subject
→ Teacher
→ Syllabus Plan
→ Daily Logs
→ Student Events

2. Discipline KPI Flow:
Total Discipline Issues
→ Class
→ Student
→ All incidents
→ Teacher remarks
→ Follow-up history
→ Resolution status

3. Teacher Compliance Flow:
Teacher Compliance %
→ Teacher
→ Missing Logs
→ Specific Periods
→ Exact Daily Log Entries

4. Student Performance Flow:
Achievements Recorded
→ Class
→ Student
→ Achievement timeline
→ Supporting remarks/events

5. Coordinator Monitoring Flow:
Pending Verifications
→ Coordinator
→ Assigned Classes
→ Pending Logs
→ Exact log awaiting verification

6. Future Dairy Example:
Low Milk Yield
→ Batch/Animal Group
→ Individual Animal
→ Feed History
→ Health Events
→ Worker Entries

7. Future Nursery Example:
Plant Mortality
→ Category
→ Batch
→ Plant Group
→ Propagation Cycle
→ Daily Entries

MANDATORY RULES:

1. Every KPI card should support click-through where logically possible.
2. Every table row should open detailed contextual data.
3. Every alert should link to source records.
4. Every graph/chart should support interactive drill-down.
5. Every summary should connect to raw operational entries.
6. Avoid dead-end screens.
7. Avoid isolated static reports.
8. Maintain breadcrumb navigation throughout drill-down hierarchy.
9. Add global search and contextual filters.
10. Maintain drill-down performance optimization for large datasets.

UI/UX REQUIREMENTS:

- Use breadcrumb navigation.
- Use expandable/collapsible panels where useful.
- Use side drawers/modals for quick detail inspection.
- Use clickable cards and smart navigation.
- Preserve filters during drill-down.
- Allow quick return to previous level.
- Add contextual summaries at each level.

The system should feel like:
an operational intelligence command center,
NOT a static ERP.

This drill-down philosophy is mandatory across current and future modules:
- Academic Monitoring
- Admissions CRM
- DairyPro
- Nursery Operations
- SancharPro

All future architecture and UI decisions must align with this drill-down operational visibility philosophy.
